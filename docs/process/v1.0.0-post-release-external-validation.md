# v1.0.0 Post-Release External Validation

## Purpose

Validate the sealed `v1.0.0` tag from a clean external clone after the final release tag was created.

This record is intentionally post-release evidence. It does not move or rewrite the `v1.0.0` tag.

## Clone

```bash
git clone --branch v1.0.0 /Users/abab/Desktop/Agent-Orchestratoar /private/tmp/agent-orchestrator-v1-final-eval-20260529
```

Observed:

- Clone exited 0.
- The annotated tag resolved to commit `9504388188add84c5887cf96886cb524087fe1f5`.
- The clone checked out detached HEAD at the release commit.
- `git status --short`: clean before validation commands.

## Validation Commands

```bash
PYTHONPATH=src python -m agent_orchestrator.cli health --format json
PYTHONPATH=src python -m agent_orchestrator.cli team setup --runtime command --format json
PYTHONPATH=src python -m agent_orchestrator.cli team workspace-status --format json

PYTHONPATH=src python -m agent_orchestrator.cli team governance-bundle inspect .agent_orchestrator/governance/v1.0.0-external-clone-bundle.json --format json
```

## Observed Results

- `pyproject.toml` reports package version `1.0.0`.
- README points to `docs/releases/v1.0.0.md` and `docs/process/v1.0.0-evidence-packet.md`.
- `health --format json`: exit 0; `codex`, `claude`, and `mock` provider states visible.
- `team setup --runtime command --format json`: exit 0; `release_readiness.ready: true`; package version `1.0.0`; provider evidence summary format `agent_orchestrator.provider_evidence_summary.v1`.
- Fresh clone runtime measurement reported `measurement_status: unavailable` because there are no local job records; `rc_blocking: false`.
- `team workspace-status --format json`: exit 0; format `agent_orchestrator.workspace_index.v1`; dirty state false.
- `team evidence-gates --format json`: exit 0; format `agent_orchestrator.evidence_bundle.v1`; status `ready`.
- `team check-compliance`: exit 0; status `passed`; blocking false.
- `team governance-bundle export`: exit 0; wrote `agent_orchestrator.governance_bundle.v1`.
- `team governance-bundle inspect`: exit 0; complete true; auditable true; blocking false; no warnings.

## Boundary Confirmation

The external clone preserves the v1 boundary:

- Governance export and inspection are read-only.
- Provider-owned refs remain evidence only.
- Token/cost remains placeholder unless reported directly by a provider runtime.
- Fresh-clone absence of local runtime history is non-blocking when the measurement surface is present.

## Result

`v1.0.0` validates from a clean external clone as a local-first externalized governance release.
