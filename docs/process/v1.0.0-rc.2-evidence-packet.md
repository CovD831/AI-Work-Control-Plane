## Known Boundaries

- Token and cost values remain placeholders unless a runtime reports them directly.
- A fresh clone with no job history can report `runtime_measurement.measurement_status: unavailable`; this is expected and not RC blocking when the surface is available and the readiness payload explains the gap.
- `send` and `cancel` remain local job/session controls, not provider-native interactive session guarantees.
- The candidate does not implement a full Codex/Claude bridge, persistent provider session manager, plugin marketplace, or direct-API patch engine.
